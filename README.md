# unofficial-codex-wiki

`unofficial-codex-wiki` is an unofficial private local mirror of selected OpenAI Codex documentation for offline reading, local search, and AI-agent context ingestion.

This project is not affiliated with, endorsed by, or maintained by OpenAI.

## Setup

Use Node.js 24 or newer and pnpm through Corepack:

```bash
corepack enable
pnpm install
pnpm typecheck
pnpm test
```

SQLite search uses `better-sqlite3`. On a fresh machine, native build prerequisites may be needed for your OS. If sandboxed home caches are read-only, use writable npm/node-gyp cache directories while rebuilding the native module.

## Local Mirror Workflow

Run the full local pipeline:

```bash
pnpm docs:sync
```

Run stages individually:

```bash
pnpm docs:discover
pnpm docs:fetch
pnpm docs:transform
pnpm docs:chunk
pnpm docs:index
pnpm docs:validate
```

Use local outputs:

```bash
pnpm docs:search "codex"
pnpm docs:read agent-approvals-security
pnpm docs:read agent-approvals-security#sandbox-and-approvals
```

Important generated outputs:

```text
data/latest/manifest.json
data/latest/validation-report.json
data/latest/diff.json
generated/markdown/codex/
generated/agent/docs.pages.jsonl
generated/agent/docs.chunks.jsonl
generated/agent/docs.manifest.json
generated/search/docs.sqlite
```

## Validation

`pnpm docs:validate` is intentionally strict. It fails if discovery lists pages that are missing from the manifest, generated Markdown, JSONL records, chunks, or search index.

If you previously ran a limited smoke command such as `--limit 1`, validation may fail because the local mirror is incomplete. Run a full `pnpm docs:sync` to produce a complete validated mirror.

V1 will build a deterministic local documentation mirror:

```text

-> Markdown-first page fetch
-> generated Markdown
-> agent JSONL pages and chunks

```

V1 must remain CLI-first, private-use-only, and cross-platform for Ubuntu/Linux, Windows, and macOS.

## Exclusions

V1 intentionally does not include MCP, embeddings, vector database, chatbot, local API, local viewer, scheduled sync, public publishing, or AI-generated summaries.
